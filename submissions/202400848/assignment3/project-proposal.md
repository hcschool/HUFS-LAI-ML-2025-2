# IRAC 자동 분류기



## 1. 해결하고자 하는 문제
본전공인 국제통상학과 과목 중 법 과목들은 시험에서 만족할 만한 점수를 얻기 위해서는 교재를 읽어야 한다. 그러나 교재 중 모든 내용을 알아야 하는 것은 아니고, 교재 중 I(Issue), R(Reasoning), A(Analysis), C(Conclusion) 부분만 알면 된다.
매주 영어로 된 교재의 모든 내용을 읽기에도 번거롭지만 IRAC에 해당하는 내용들을 미리 정히해 두려면 시간이 더 많이 걸린다. 교재 pdf를 넣으면 자동으로 수업시간에 다룬 케이스의 IRAC만 분류해서 알려주는 모델을 제작해 보려 한다. 평소에 언젠가 만들어 봐야겠다고 생각했던 기능인데 이번 기회에 제작해 봄으로써 시간을 효율적으로 사용하고 싶다.

## 2. 제안하는 솔루션
교재의 pdf를 넣으면 텍스트를 인식해서 각 케이스의 I, R/A(보통 교재에서는 reasoning, analysis가 명확히 구분되어서 나오지 않음), C의 3개 클래스를 구분해서 알려주는 모델을 제작할 게획이다. 가능할지는 모르겠지만, 단순히 문장을 알려주는 것보다는 pdf에 밑줄을 쳐 주는 등 시각화가 가능할 수 있으면 더 편리한 사용이 가능할 것 같다.

## 3. Assignment별 계획

### Assignment 4: 데이터 수집 및 분석
- 데이터는 기본적으로 지난 세 학기동안 본인이 구분해 둔 I, R/A, C 문장들을 가지고 학습시킬 계획이다. (정확도 개선을 위해 작업해둔 것들을 제미나이로 검토할 계획이다.) 데이터셋은 교재의 문장을 복사 붙여넣기로 라벨링한 후 json형식으로 저장할 계획이다.
- 1주 당 생성 가능한 데이터의 양 (양이 적은 주차 기준):
  - (I, R/A, C, 총합): (3, 17, 4, 24), (5, 9, 8, 22) 정도.

### Assignment 5: 모델 학습 및 평가
- 우선, 수강했던 세 과목에서 한 과목당 5주차씩 학습시켜 총 15주차 데이터셋을 가지고 모델을 학습시킨다.
- 만일 정확도가 낮다면 전체 자료로 데이터셋을 만들어 학습시킨다.
- 만일 그래도 정확도가 낮다면 data augmentation을 사용해 볼 생각이다.
- 평가는 이번 학기에 수강중인 과목 중 한 주차를 사용할 것이다.
- 사용할 모델 후보: Legal-BERT, RoBERTa-Large, DeBERTa-v3-base, DistilBERT 중 사용하려 계획중이다.
- I, R/A, C 클래스별로 precision, recall, macro f1-score(클래스별로 데이터 개수 편차가 있기 때문)을 계산하고 전체 클래스에 대해서도 구할 것이다.
