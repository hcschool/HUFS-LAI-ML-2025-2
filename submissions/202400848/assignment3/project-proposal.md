# 문맥 기반 대체 어휘 추천 시스템

## 해결하고자 하는 문제
영어 라이팅 연습을 하고 있다. 그러나 평소에 글을 읽을 때 내가 아는 단어를 영작할 때 잘 사용하지 못하고, 익숙한 쉬운 단어만 사용하는 경향이 있다. 시중 무료 작문 교정기나 ai에게 시키면 내가 쓴 문장을 문법 오류가 없어도 자꾸 바꾸어 주는데, 사용하다 보니 내가 ai의 문체를 닮아간다고 생각하게 되었다. 또한 매 단어마다 일일이 thesaurus를 찾는 것도 시간이 오래 걸린다. 사람같은 글쓰기 스타일을 유지하면서도 사용하는 단어의 폭을 넓히기 위해 이 프로젝트를 추천하게 되었다.

## 해결 방법
- 사용 방법 및 목적
  - 입력: 내가 영작한 내용을 붙여넣기 한다.
  - 출력: 모델이 내가 영작한 내용을 보고 내가 사용한 단어의 유의어를 추천해 준다.
  - 목적: 영작 시 단어 사용의 폭을 늘리고 고급 어휘 사용을 도와주기 위해.

## 세부 계획

### Assignment 4
- 데이터셋: WikiText-103 공개 데이터셋
- wordfreq 라이브러리로 zipf’s score이 3.0 미만인 단어를 검색한다. (사용 빈도가 적은 단어 = 더 어려운 단어)
-  WordNet에서 그 단어의 유의어를 찾고, zipf’s score가 5.0 이상인 쉬운 단어로 바꾸는data augmentation을 사용한다.
- 이렇게 단어를 바꾼 데이터셋에서 단어를 바꿔도 어색하지 않은 문장만 골라낸다.
  - 아마 데이터 전처리 방식은 POS Tagging으로 품사 확인, SBERT 사용해 유사도 검사를 사용할 것 같다.

### Assignment 5
- 모델: BERT, SBERT, DistilBERT 중 데이터셋 크기와 모델 성능을 고려해 학습용 모델과 평가용 모델을 설정.
- Assignment4를 사용한 데이터셋을 학습시킨다.
- Evaluation
  - Top-k accuracy: 모델이 추천한 상위 n개 안에 정답인 단어가 포함되어 있는가?
  - SBERT의 코사인 유사도를 측정해 바꾼 단어가 원래 문장과 의미가 달라졌는지 여부를 확인한다.

### Assignment 6
- 시각화 방법은 생각중이다. html웹사이트를 만들어 내가 붙여넣기한 문장에서 고칠 단어에 밑줄을 쳐 두어 클릭하면 보여주는 방식을 사용할 것 같다. (더 나은 방법을 생각중)
- Assignment5의 성능이 기대 이하이면 공개api를 사용할 것이다.
