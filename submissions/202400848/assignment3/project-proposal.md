# 학교 홈페이지 통합 알림 봇

## 해결하고자 하는 문제
내가 학교 행사나 중요한 공지사항을 확인하기 위해서는 크게 외대 메인 홈페이지, 본전공 홈페이지, 이중전공 홈페이지 등의 여러 페이지를 방문해야 한다. 그러나 매번 다른 사이트를 여러번 확인하기에는 시간이 낭비라고 느껴졌다. 또한 메인 홈페이지의 경우 HUFS공지, 학사공지, 장학공지를 매번 열어보며 확인하기 불편했다. 메인 홈페이지와 학과 공지가 중복되는 내용도 있고 카테고리 구분이 불명확한 것이 있기도 해서 저번에 본 공지인지 확인하기도 불편하다. 나에게 필요한 사항을 카테고리별로 분류해서 중복되지 않게 내게 필요한 것만 알려주는 모델을 만들어 해결하겠다.

## 해결 방법
- 사용 방법 및 목적
  - 입력: 학교 여러 홈페이지의 공지사항 내용들.
  - 출력: 일정 기간 간격으로 알림을 발송한다.
  - 목적: 중복되지 않는 학교 전체 공지사항을 개인 맞춤형으로 안내받아 중요한 공지를 놓치지 않게 하기 위해서이다.

## 세부 계획

### Assignment 4
- 데이터셋: 학교 메인 홈페이지 중 HUFS 공지, 장학 공지, 학사 공지, 본전공과 이중전공 홈페이지 등 사이트의 데이터 크롤링.
- 학교 사이트에서 관련 공지사항을 분리하고 카테고리를 분류한다.
- 공지사항의 내용을 전부 또는 일부 포함할 것이다 (아마 약 첫 512토큰 정도가 될 것 같지만 변경 가능하다)
- 제목과 기한, 공지 유형 등을 라벨링하고 중복되는 공지사항을 제거한다.
- 나와 관련 있는지 여부를 직접 라벨링한다.

### Assignment 5
- 모델: 아마 KoBERT를 사용할 것 같다(데이터셋 양이나 성능에 따라 변경 가능)
- Assignment4를 사용한 데이터셋을 학습시킨다.
- Evaluation
  - 카테고리가 분명하게 나뉘어 있지 않은 공지사항의 경우: Accuracy, F-1 Score 등을 활용하여 카테고리를 제대로 분류했는지 확인한다.
  -  나와의 관련도 예측을 잘 했는지: 직접 train, validation에 사용하지 않은 데이터셋으로 precision, recall 등을 사용한다.

### Assignment 6
- 일정 기간을 두고 이메일 (또는 노션 또는 다른 내가 자주 사용하는 다른 기능으로 생각중)으로 알림을 발송한다.
- 마감 기간이 3일 이내로 남은 경우이거나 선착순인 경우 즉시 이메일로 알림을 보낸다.
- 또는 나만의 공지사항 확인용 탭을 시각화해서 여기로 알림을 전달하게 할 수도 있다.
